## Summary

Implemented QUIC/HTTP3-aware connection pooling so the HostPool and ConnectionActor
distinguish between "new connection needed" (TCP/TLS for HTTP/1.x, HTTP/2) and
"new stream on existing connection needed" (QUIC for HTTP/3).

## Key Design Decisions

1. **FakeConnectionActor forwarding**: Added `ReceiveAny` forwarding to controlProbe so QUIC tests can observe `OpenNewStream` messages sent to the fake connection actor.

2. **HandleConnectionReady split**: Non-QUIC flushes all pending requesters with shared handle (preserving existing behavior). QUIC uses `ServeQueuedRequesters()` for version-aware per-stream routing.

3. **Suppress extra QUIC connection spawns**: When a QUIC connection is still connecting (no handle yet), don't spawn additional connections — just queue requesters. Only spawn new connections when all existing QUIC connections are at capacity.

## Issues Encountered

1. **Build errors**: FakeConnectionActor returned `IActorRef`, but QUIC tests called `ExpectMsg` (a TestProbe method) on it. Fixed by adding message forwarding and observing on controlProbe.

2. **EH-003 regression**: Changing `HandleConnectionReady` to use `ServeQueuedRequesters` broke non-QUIC slot-tracking semantics. Fixed by keeping the old flush-all behavior for non-QUIC.

3. **QUIC-006 extra spawns**: `HandleEnsureHost` spawned extra connections while QUIC was still connecting. Fixed by checking for handle-less active connections before spawning.

4. **QUIC-007 message ordering**: The 101st QUIC request spawned a new connection whose `Self` arrived at controlProbe before the `OpenNewStream` from `StreamCompleted`. Fixed by draining the expected `IActorRef` before asserting `OpenNewStream`.
